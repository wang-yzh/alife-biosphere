# Terminology Index v1

## Purpose

This is the first shared terminology index for `alife_biosphere`.

It does not try to define every term from biology, ecology, and artificial
life. It only defines the recurring project terms that appear often enough to
create ambiguity if left implicit.

Each entry gives:

- the intended project meaning
- what the term should not be confused with

This is a coordination tool, not a full ontology.

## Core Terms

## `archive`

Project meaning:

- the population- or clade-level store of transmissible capsules, subject to
  visibility limits, validation, and forgetting

Do not confuse with:

- a full model checkpoint store
- unlimited shared memory

## `capsule`

Project meaning:

- a compact transmissible unit such as a protocol fragment, heuristic, caution
  pattern, or signaling convention fragment

Do not confuse with:

- a complete policy
- a raw replay trace

## `connectedness`

Project meaning:

- a profile of structural coupling summaries such as interaction density,
  module separation, corridor concentration, and redundancy

Do not confuse with:

- one scalar edge count
- modularity alone

## `development`

Project meaning:

- the costly shaping of phenotype during the juvenile period through exposure,
  plasticity, and limited inherited bias

Do not confuse with:

- free pretraining
- ordinary adult adaptation

## `direct trust`

Project meaning:

- trust formed through repeated direct interaction with a partner or local role

Do not confuse with:

- archive-source trust
- global public reputation

## `disturbance`

Project meaning:

- an explicit perturbation process such as local crash, regional stress,
  resource collapse, parasite bloom, or signal pollution

Do not confuse with:

- generic random noise
- any short fluctuation

## `ecological memory`

Project meaning:

- persistent world modifications inherited indirectly through habitat traces,
  depletion, signals, or altered local conditions

Do not confuse with:

- private organism memory
- cultural archive entries

## `evidence`

Project meaning:

- logged observations or experiment results that can support or weaken a claim

Do not confuse with:

- mechanism ideas
- theoretical plausibility

## `founder event`

Project meaning:

- a small colonizing migration or branching event whose scientific meaning
  depends on later persistence and altered selection

Do not confuse with:

- any movement between habitats

## `higher-level individuality`

Project meaning:

- a stronger status than grouping or cooperation, requiring evidence that the
  group has become a meaningful unit of persistence and reproduction

Do not confuse with:

- co-location
- functional grouping
- group persistence alone

## `inheritance channel`

Project meaning:

- one route by which structure persists across time, such as genetic,
  developmental, somatic, cultural, or ecological inheritance

Do not confuse with:

- one generic memory stream

## `kin group`

Project meaning:

- a group tied by lineage anchor, local interaction, and possible shared
  resource or signaling context

Do not confuse with:

- any spatial cluster
- a fully realized higher-level individual

## `lineage`

Project meaning:

- the ancestry-linked continuation of structure across generations or founder
  branches

Do not confuse with:

- current habitat occupancy
- species label

## `novelty`

Project meaning:

- the appearance of new behaviors, structures, or organizational forms relative
  to relevant prior context

Do not confuse with:

- raw randomness
- instability by itself

## `open-endedness`

Project meaning:

- a plural family of long-run evolutionary properties involving continued
  novelty, restructuring, complexity change, or ecological expansion

Do not confuse with:

- long runtime
- high variance
- raw diversity alone

## `provenance`

Project meaning:

- the origin and derivation history of a capsule or other transmitted artifact

Do not confuse with:

- simple source id only

## `refuge`

Project meaning:

- a habitat or structural pathway that supports persistence and rescue under
  disturbance

Do not confuse with:

- a cost-free safe zone

## `regime shift`

Project meaning:

- a persistent change in organization that alters later selection or
  coordination, not merely a short transient release

Do not confuse with:

- any abrupt change
- any adaptive-cycle release phase

## `rescue`

Project meaning:

- a reversal from decline toward persistence, attributable to one or more
  classified sources such as retained variation, migration, inheritance, or
  cultural import

Do not confuse with:

- ordinary short rebound noise

## `source trust`

Project meaning:

- trust attached to archive or broadcast sources in a domain-sensitive way

Do not confuse with:

- direct trust in a partner from repeated local interaction

## `warning candidate`

Project meaning:

- a typed, uncertain diagnostic object indicating collapse risk,
  resilience loss, regime-shift candidate, or recovery opportunity

Do not confuse with:

- a truth label
- a guaranteed tipping-point detector

## Working Style Note

When possible, project writing should prefer these terms consistently, rather
than swapping between near-synonyms without comment.

The main reason is simple:

```text
coordination errors often start as terminology errors.
```
