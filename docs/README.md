# Alife Biosphere Documentation Library

## Purpose

This file is the entry map for the `alife_biosphere` documentation library.

The repository has grown through multiple reading passes, design revisions, and
handoff notes. The goal here is not to rewrite every historical document. The
goal is to say clearly:

- what is current
- what is supporting material
- what is historical or superseded

No documents were moved during this cleanup pass.
Organization is provided through this index and through status notes on drifted
historical files.

## Background Review Layer

Most topic-specific literature documents belong to the background review layer.

Use these naming rules:

- `*_review_v0`: topic reviews and literature synthesis notes
- `*_bibliography_*`: bibliographic support
- `literature_*`: gap analyses, backlog, or synthesis helpers

These files are important as support, but they are not all current control
documents.

## Historical Or Superseded Planning Notes

These files are intentionally kept, but they should not be read as the current
project contract:

- `artificial_life_positioning_from_classic_papers.md`
- `world_design_v0.md`
- `build_plan_v0.md`
- `repository_architecture_v0.md`

Use them for project history, discarded branches, and design lineage.
Use the ecology-first current docs for current implementation decisions.

## Naming Conventions

- `*_v1`: current coordinated documents or upgraded notes
- `*_v2`: current major realignments of core design or build planning
- `*_spec`: implementation-grade scope freezes
- `*_ledger_*`: evidence and result registers
- `*_table_*`: project control tables
- `*_register_*`: tracked cautions, contradictions, or open questions
- `*_review_v0`: background topic reviews, usually not implementation
  contracts
