# Ant Sandbox M14 Niche Substrate Spec v1

## Purpose

M14 adds environmental substrates that can become future ecological niches.

The current sandbox has ants, food, terrain, trails, death, reproduction, and
checkpointed branches. That is enough for stable ant ecology, but not enough for
open-ended ecological expansion.

The next world layer must let ant activity create new opportunities.

## Core Rule

Do not add a new species first.

Add a substrate that a future species could exploit.

The correct order is:

```text
ant activity
-> persistent environmental byproduct
-> measurable ecological opportunity
-> successor life layer
```

## Candidate Substrates

M14 can introduce one or more of these, but should start with one:

- corpses from dead ants
- waste or residue near nests and food routes
- food fragments dropped during transport
- terrain disturbance from heavy traffic
- long-lived trail-conditioned soil
- seasonal resource pulses
- food classes with different handling costs

## Recommended First Substrate

The recommended first substrate is:

```text
corpses and residue
```

Reason:

- death already exists
- ant traffic already creates spatial concentration
- corpse and residue fields can later support decomposers, fungus, scavengers,
  or pathogen pressure
- this connects directly to the user's open-evolution example of one organism's
  waste becoming another organism's resource

## M14A Corpse Field

Add a persistent corpse object when an ant dies.

Fields:

- `corpse_id`
- `x`
- `y`
- `colony_id`
- `death_tick`
- `energy_value`
- `decay_ticks_remaining`
- `death_reason`

Behavior:

- corpse appears at death location
- corpse decays over time
- corpse is visible in observer
- corpse is serialized in checkpoints
- corpse count is included in branch comparison

Initial restriction:

- ants do not need to consume corpses yet
- no decomposer organism yet

Acceptance:

- deaths leave visible short-lived ecological material
- corpse distribution reflects actual death geography

## M14B Residue Field

Add local residue where ants unload, feed, or repeatedly traverse.

Fields:

- cell position
- residue value
- source type: `nest`, `food`, `trail`, `corpse`
- decay rate

Behavior:

- residue accumulates near dense activity
- residue decays slowly
- residue can be rendered as a subtle substrate layer
- residue is serialized in checkpoints

Initial restriction:

- residue should not directly improve ant behavior unless explicitly tested

Acceptance:

- long-running branches show different residue maps
- residue can become input to M16 successor life

## M14C Food Classes

Introduce typed food patches.

Possible classes:

- `sugar`: easy pickup, low value
- `seed`: slower handling, high nest value
- `protein`: rare, supports reproduction or future larvae
- `fungal`: future bridge to decomposer/fungus systems

Initial behavior:

- same pickup loop
- different value score
- optional different handling delay

Acceptance:

- branches can diverge by dominant food class
- food class identity appears in observer and reports

## Data Model Requirements

Any new substrate must:

- have explicit state objects or fields
- serialize through checkpoint
- appear in observer or report
- have summary metrics
- have deterministic behavior under seed control

## Output Metrics

M14 should report:

- corpse count
- residue cell count
- residue total value
- food class remaining totals
- food class pickup totals
- substrate distribution by colony territory
- substrate persistence duration

## Tests

Required tests:

- corpse appears when an ant dies
- corpse decays deterministically
- substrate state survives checkpoint round-trip
- observer payload includes substrate layer
- summary reports substrate counts

Targeted test path:

```text
tests/test_ant_sandbox_niche_substrate.py
```

## Acceptance Criteria

M14 is complete when:

- at least one substrate is generated by ant activity
- the substrate persists long enough to matter
- branch comparison can report substrate differences
- the substrate is visible or inspectable
- no successor species is needed to justify the substrate

## Failure Modes

Stop if:

- substrate is purely decorative
- substrate is globally applied rather than locally produced
- substrate cannot be saved in checkpoints
- substrate has no metric
- substrate immediately becomes another hard-coded food source without a clear
  ecological role

## Execution Note

M14 is the first true bridge from ant sandbox to ecological open-endedness.

It should be implemented conservatively.

The first success is not "new species".

The first success is:

```text
the ants leave behind a world that future life could exploit.
```
