# Ant Sandbox Open Evolution Engineering Plan v2

## Purpose

This plan starts after `M17`.

That means the branch already has:

- a stable ant sandbox substrate
- checkpoint and fork runtime
- observer and branch comparison
- niche substrate from corpse and residue
- first successor life via weak decomposer patches
- multi-branch campaign orchestration

The question is no longer:

```text
can one successor layer depend on ant-created substrate?
```

The question now is:

```text
how do we move from one dependency edge
to multiple interacting ecological chains
without losing interpretability?
```

## Why v2 Exists

`ant_sandbox_open_evolution_engineering_plan_v1.md` completed its intended
route:

```text
M12 -> M13 -> M14 -> M15 -> M16 -> M17
```

That plan built the experimental method.

This plan defines the next mainline:

```text
economic stabilization
-> second successor ecology line
-> longer dependency chain
-> broader campaign coverage
-> candidate third niche introduction
```

## Core Interpretation

Open evolution in the next phase should not mean:

- more mutation by default
- more organisms for spectacle
- faster visual complexity
- bigger dashboards
- more combat before ecology deepens

It should mean:

```text
new ecological structures create the conditions for still newer structures
and forked branches begin to specialize into meaningfully different worlds
```

## Current Limitation

Right now the branch has one realized dependency chain:

```text
ant death/activity
-> corpse / residue
-> decomposer
-> enriched residue
```

That is a real milestone, but still a short chain.

The next phase should extend this into a second ecological line instead of
stacking more complexity onto ants themselves.

## Main Rule

Every post-`M17` feature must satisfy all of these:

- visible in observer
- checkpoint-safe
- branch-comparable
- reflected in open-endedness metrics with honest status labels
- ecologically dependent on prior state, not decorative

If any one of these fails, the feature should not be treated as part of the
mainline.

## Master Milestone Order

The new phase is:

```text
M18 economic stabilization
M19 fungus layer
M20 dual dependency-chain campaign
M21 broad campaign matrix
M22 third niche candidate gate
```

This ordering is intentional.

We should stabilize the substrate before adding a second successor layer, then
use campaigns to test whether the new chain actually changes branch history.

## M18 Economic Stabilization

Goal:

```text
reduce late-stage colony collapse pressure enough that upper ecological layers
are not built on a brittle baseline
```

Reason:

- current late-stage deaths are still dominated by starvation
- population often slides downward in the later horizon
- successor layers should not be evaluated on top of a collapsing colony floor

Scope:

- tune population ceiling versus initial population
- tune upkeep / metabolism balance conservatively
- preserve resource pressure
- avoid explosive population growth

Preferred tuning order:

1. `max_population`
2. upkeep or metabolism
3. lifespan only if needed

Acceptance:

- later-horizon population decline is less severe
- birth/death balance remains bounded
- pressure remains visible
- no uncontrolled population explosion

What not to do:

- do not simply inflate food everywhere
- do not zero out starvation pressure
- do not tune until all branches look identical

## M19 Fungus Layer

Goal:

```text
introduce a second successor ecology line that depends on residue or enriched
residue rather than directly on ants
```

Reason:

- fungus is a better next step than a mobile scavenger
- it extends the existing decomposer chain rather than replacing it
- it keeps the system patch-based and readable

Preferred dependency:

```text
residue / enriched residue
-> fungus patch
```

Desired result:

```text
ant activity
-> corpse / residue
-> decomposer
-> enriched residue
-> fungus
```

Scope:

- fungus patch emergence
- fungus growth and decay
- weak spread through local enriched residue
- observer rendering
- checkpoint support
- branch comparison support
- open-endedness metrics support

Acceptance:

- fungus cannot appear without prior substrate
- fungus leaves a different spatial signature from decomposers
- at least one branch shows persistent fungus occupancy

What not to do:

- do not make fungus a food item yet by default
- do not make fungus a complex agent
- do not let fungus dominate the entire map immediately

## M20 Dual Dependency-Chain Campaign

Goal:

```text
run campaigns where two successor layers can shape branch history
```

This is the first phase where we should seriously ask whether different
branches are becoming different ecological worlds rather than merely different
ant logistics outcomes.

Campaign should compare:

- ants only
- ants + decomposer
- ants + decomposer + fungus

Key outputs:

- branch observers
- branch comparison
- open-endedness metrics
- campaign summary

Acceptance:

- at least one branch specializes toward stronger decomposer signal
- at least one branch specializes toward stronger fungus signal
- differences are visible in observer and not only in counters

## M21 Broad Campaign Matrix

Goal:

```text
broaden coverage so conclusions are not based on one small family of branches
```

Scope:

- more fork seeds
- longer horizons
- selected environment perturbations
- selected inheritance-mode conditions

Recommended matrix dimensions:

- root seed
- fork seed
- successor-layer toggle
- one or two pressure conditions

Acceptance:

- campaign results are summarized across multiple families
- branch specialization remains visible beyond one hand-picked run
- output remains organized enough to inspect manually

What not to do:

- do not explode the matrix before summaries are readable
- do not add dimensions faster than reporting can keep up

## M22 Third Niche Candidate Gate

Goal:

```text
decide whether the next ecological line should be scavenger, plant/seed, or
another patch-based layer
```

This is a gate, not an automatic implementation.

The branch should choose the third line only after M19-M21 show real
specialization pressure.

Candidate directions:

- `scavenger`
  Strength: directly consumes corpse or abandoned food.
  Risk: introduces a second mobile-agent system too early.

- `plant/seed`
  Strength: creates a new long-timescale resource chain.
  Risk: larger ecological model jump.

- `fungal specialization extension`
  Strength: deepens the existing residue chain with lower implementation risk.
  Risk: may delay diversity of organism forms.

Acceptance:

- choice is justified by campaign evidence
- choice preserves observer/comparison/metrics discipline
- choice is not made for visual novelty alone

## What To Avoid In v2

Avoid these dead ends:

- more mutation before ecological demand increases
- more combat before successor ecology branches
- many new organisms at once
- agent-heavy successor layers before patch layers are exhausted
- metrics inflation without new ecological meaning

## Deliverables Rule

Every milestone in v2 must include:

- implementation code
- runnable script
- observer-visible outcome
- branch comparison impact
- open-endedness metric impact
- slice note
- updated status page

## One-Sentence Plan

The post-`M17` mainline is:

```text
stabilize the colony economy,
grow a second successor ecology line,
and use broader campaigns to see whether branch histories diversify into
multiple interacting niche systems.
```
